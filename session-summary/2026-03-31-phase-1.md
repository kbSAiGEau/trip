## Confirmed Requirements (Phase 1 Only)

### Echo Check
- **Key Deliverable:** A personal PWA trip companion that shows itinerary, transit, and accommodation info organized by destination section with a glance-first "today" view
- **Critical Fact:** Must work offline (PWA with service worker) — user will be in Swiss Alps, on ferries, and in transit between countries
- **Hardest Problem:** Timezone handling across 4 timezone changes (AEST → CET → EEST → TRT) with correct local time display per destination

### Blueprint
- **Architecture:** Next.js PWA with static site generation, single trip data file, date-aware component rendering
- **Tech Stack:** Next.js, TypeScript, Vercel, PWA (service worker)
- **Top Risks:**
  1. iOS PWA limitations — mitigated by static display (no push/background sync needed)
  2. Timezone bugs — mitigated by explicit timezone storage in data file
  3. Offline cache staleness — mitigated by cache-first strategy with manual refresh
- **Complexity:** S/M (small-medium — static data app with smart presentation layer)

### Requirements Summary

**Core Features (MVP):**
- 4 trip sections with per-section color theming on light base
- Cumulative day counter (Day 1 = departure from Sydney)
- Daily activity highlights
- Transit cards (flights, trains, ferries) with times in local timezone and app notes
- Accommodation block per section (hotel name + address)
- Glance-first "today" view (default screen)
- "Next transit" prominently surfaced
- Scrollable full-trip timeline

**Out of Scope:**
- Live APIs (flight tracking, weather)
- Authentication
- Push notifications
- Budget tracking, packing lists
- Multi-user sharing

**User Journeys:**
1. "What's happening today?" — open app, see today's section/day/highlights
2. "What's my next transit?" — find upcoming flight/ferry/train with details
3. "Overview browse" — scroll full trip timeline

**Data Model:**
- Trip → Sections → Days → Activities + Transit events
- Accommodation per section
- All data in single editable file
- Placeholder data (trip not finalized)

**Design:**
- Glance-first, tap to expand
- Per-section color theming (Swiss red, Tomorrowland purple, Greek blue, Turkish turquoise)
- Light cream/off-white base

**Technical Decisions:**
- Local time per destination
- Travel days belong to departing section
- Day 1 = departure day from Sydney
- No auth, obscure URL
- PWA offline-capable
